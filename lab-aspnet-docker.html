Cài đặt package postgresql trong dự án ASP.NET

Mở Package Manager Consoleđể cài package này cho tất cả các dịch vụ gồm: StudentService, TeacherService.

Các bạn sẽ thay đổi dịch vụ tại Default project. Giả sử: Default project đang là StudentService thì nó sẽ cài package này cho StudentService mà thôi.

Trong lệnh dưới: x.x.x là phiên bản cần cài. Các bạn phải check xem EntityFrameworkCore mà bạn đang sử dụng là version nào thì sẽ dùng tương ứng với x đầu tiên.

Ví dụ: EntityFrameworkCore là 8.0.25 thì x.x.x sẽ là 8.0.0. AspNetCoreMicroservicesDemo and two services such as StudentService and TeacherService.

Example (EF Core 8):

Example 
Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version 8.0.0

Cập nhật Connection String in appsettings.json

Cần phải cập nhật tất cả appsettings.json cho StudentService, TeacherService.

Lấy ví dụ với StudentService, trong ConnectionString, thay thế giá trị của “DefaultConnection”.

Từ thuộc tính ConnectionString này:

Example (trong StudentService/appsetting.json):

JSON — appsettings.json (local dev) 
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=microservicesdb;Username=postgres;Password=postgres"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
1

Install the Npgsql.EntityFrameworkCore.PostgreSQL Package

Via Package Manager Console in Visual Studio
You need to install this package for each service — StudentService and TeacherService. Repeat the command by switching the Default project in the Package Manager Console.
1a. Open Package Manager Console

In Visual Studio: Tools → NuGet Package Manager → Package Manager Console

1b. Select the target project

In the Default project dropdown (top of the console), choose StudentService first:

1c. Run the install command
Replace x.x.x with the correct version. The first digit must match your EF Core version. Example: EF Core 8.0.25 → use 8.0.0.
Package Manager Console 
Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version x.x.x

Example (EF Core 8):

Example 
Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version 8.0.0
1d. Repeat for TeacherService

Change the Default project to TeacherService and run the same command again.

Check your current EF Core version in .csproj: look for Microsoft.EntityFrameworkCore package version to determine the correct first digit.
2

Create the docker-compose.yml File

In the root of your cloned repository

Create docker-compose.yml in the AspNetCoreMicroservicesDemo folder (the root folder you pulled from GitHub).

File location: AspNetCoreMicroservicesDemo/docker-compose.yml
docker-compose.yml
YAML 
version: '3.8'

services:

  postgres:
    image: postgres:16
    container_name: microservices-postgres
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: microservicesdb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  studentservice:
    build:
      context: .
      dockerfile: StudentService/Dockerfile
    container_name: student-service
    ports:
      - "5001:80"
    environment:
      ConnectionStrings__DefaultConnection: "Host=postgres;Port=5432;Database=microservicesdb;Username=postgres;Password=postgres"
    depends_on:
      - postgres

  teacherservice:
    build:
      context: .
      dockerfile: TeacherService/Dockerfile
    container_name: teacher-service
    ports:
      - "5002:80"
    environment:
      ConnectionStrings__DefaultConnection: "Host=postgres;Port=5432;Database=microservicesdb;Username=postgres;Password=postgres"
    depends_on:
      - postgres

volumes:
  postgres_data:
The depends_on key ensures PostgreSQL starts before your services. The hostname postgres inside the connection string refers to the Docker service name, not localhost.
3

Configure DbContext to Use PostgreSQL

In Program.cs of each service

Open Program.cs in both StudentService and TeacherService. Replace the UseSqlServer (or in-memory) call with UseNpgsql:

Program.cs — StudentService
C# 
var builder = WebApplication.CreateBuilder(args);

// Add PostgreSQL via Npgsql
builder.Services.AddDbContext<StudentDbContext>(options =>
    options.UseNpgsql(
        builder.Configuration.GetConnectionString("DefaultConnection")
    )
);

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseAuthorization();
app.MapControllers();
app.Run();
Make sure you have using Microsoft.EntityFrameworkCore; at the top of Program.cs, otherwise UseNpgsql will not be recognized.
4

Update appsettings.json

Connection string for local development

Add a DefaultConnection entry to both services' appsettings.json. For local development (without Docker), point to localhost:

JSON — appsettings.json (local dev) 
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=microservicesdb;Username=postgres;Password=postgres"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
When running via docker-compose, the environment variable ConnectionStrings__DefaultConnection defined in docker-compose.yml automatically overrides this value.
5

Add Entity Framework Core Migration

Generate and apply database schema
5a. Add the initial migration

In Package Manager Console, set the Default project to StudentService, then run:

Package Manager Console 
Add-Migration InitialCreate
5b. Update the database
Package Manager Console 
Update-Database

Repeat for TeacherService by switching the Default project.

You can also use the .NET CLI: dotnet ef migrations add InitialCreate and dotnet ef database update from inside each service folder.
5c. (Optional) Auto-apply migrations on startup

Add to Program.cs to automatically apply pending migrations when the app starts in Docker:

C# 
// Auto-apply migrations on startup (useful for Docker)
using (var scope = app.Services.CreateScope())
{
    var db = scope.ServiceProvider.GetRequiredService<StudentDbContext>();
    db.Database.Migrate();
}
6

Run Docker Compose & Test

Start all services and verify the setup
6a. Build and start all containers

Navigate to the AspNetCoreMicroservicesDemo folder in terminal:

Terminal / PowerShell 
cd AspNetCoreMicroservicesDemo
docker-compose up --build
6b. Verify services are running
Terminal 
docker-compose ps
6c. Test the API endpoints
Student Service
http://localhost:5001/swagger
http://localhost:5001/api/students
Teacher Service
http://localhost:5002/swagger
http://localhost:5002/api/teachers
6d. Stop containers
Terminal 
docker-compose down
Use docker-compose down -v to also remove volumes (wipes the PostgreSQL data). Only do this if you want a clean slate.

Sample Dockerfile (for each service)
Dockerfile 
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 80

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["StudentService/StudentService.csproj", "StudentService/"]
RUN dotnet restore "StudentService/StudentService.csproj"
COPY . .
WORKDIR "/src/StudentService"
RUN dotnet build "StudentService.csproj" -c Release -o /app/build

FROM build AS publish
RUN dotnet publish "StudentService.csproj" -c Release -o /app/publish

FROM base AS final
WORKDIR /app
COPY --from=publish /app/publish .
ENTRYPOINT ["dotnet", "StudentService.dll"]
