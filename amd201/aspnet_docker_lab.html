1. Cài đặt package postgresql trong dự án ASP.NET

Mở Package Manager Consoleđể cài package này cho tất cả các dịch vụ gồm: StudentService, TeacherService.

Các bạn sẽ thay đổi dịch vụ tại Default project. Giả sử: Default project đang là StudentService thì nó sẽ cài package này cho StudentService mà thôi.

Trong lệnh dưới: x.x.x là phiên bản cần cài. Các bạn phải check xem EntityFrameworkCore mà bạn đang sử dụng là version nào thì sẽ dùng tương ứng với x đầu tiên.

Ví dụ: EntityFrameworkCore là 8.0.25 thì x.x.x sẽ là 8.0.0. AspNetCoreMicroservicesDemo and two services such as StudentService and TeacherService.

Example (EF Core 8):

Example 
Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version 8.0.0

1. Lab overview

This lab demonstrates how to prepare an ASP.NET Core microservice project for PostgreSQL and Docker. The example assumes a solution folder named AspNetCoreMicroservicesDemo and two services such as StudentService and TeacherService.

Scenario: You cloned a microservices solution from GitHub. Now you want to install the PostgreSQL provider, configure database access, and run the services together with PostgreSQL in Docker.

2. Prerequisites

  • .NET SDK installed
  • Docker Desktop installed and running
  • Visual Studio or Visual Studio Code
  • Basic ASP.NET Core Web API knowledge
  • Your solution already pulled from GitHub

Important The version of Npgsql.EntityFrameworkCore.PostgreSQL should normally match your major EF Core version.

4. Install the PostgreSQL package

Open Package Manager Console in Visual Studio. Make sure the Default project is set correctly before running the command.

If the Default project is StudentService, the package will be installed only for StudentService. Repeat the process for TeacherService if needed.

Example command:

Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version x.x.x

Version rule example:

  • If Microsoft.EntityFrameworkCore is 8.0.25
  • Then use an Npgsql EF Core package from the 8.x.x family

You can also use the .NET CLI:

dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL --version 8.0.0

5. ASP.NET Core sample code

5.1 Model: Student.cs

namespace StudentService.Models;

public class Student
{
    public int Id { get; set; }
    public string FullName { get; set; } = string.Empty;
    public string Email { get; set; } = string.Empty;
}

5.2 DbContext: AppDbContext.cs

using Microsoft.EntityFrameworkCore;
using StudentService.Models;

namespace StudentService.Data;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options)
    {
    }

    public DbSet<Student> Students => Set<Student>();
}

5.3 Program.cs

using Microsoft.EntityFrameworkCore;
using StudentService.Data;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseNpgsql(builder.Configuration.GetConnectionString("DefaultConnection")));

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseAuthorization();
app.MapControllers();

app.Run();

5.4 Controller: StudentsController.cs

using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using StudentService.Data;
using StudentService.Models;

namespace StudentService.Controllers;

[ApiController]
[Route("api/[controller]")]
public class StudentsController : ControllerBase
{
    private readonly AppDbContext _context;

    public StudentsController(AppDbContext context)
    {
        _context = context;
    }

    [HttpGet]
    public async Task<ActionResult<IEnumerable<Student>>> GetAll()
    {
        return await _context.Students.ToListAsync();
    }

    [HttpPost]
    public async Task<ActionResult<Student>> Create(Student student)
    {
        _context.Students.Add(student);
        await _context.SaveChangesAsync();
        return CreatedAtAction(nameof(GetAll), new { id = student.Id }, student);
    }
}

5.6 Create migration and update database

dotnet ef migrations add InitialCreate

dotnet ef database update

When using containers, many teams prefer running migrations on startup or through a separate migration step.

6. Dockerfile for StudentService

Create a Dockerfile inside each service project.

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["StudentService/StudentService.csproj", "StudentService/"]
RUN dotnet restore "StudentService/StudentService.csproj"
COPY . .
WORKDIR "/src/StudentService"
RUN dotnet publish "StudentService.csproj" -c Release -o /app/publish

FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS final
WORKDIR /app
COPY --from=build /app/publish .
EXPOSE 8080
ENTRYPOINT ["dotnet", "StudentService.dll"]

Use the same idea for TeacherService, changing the project paths and DLL name.

7. Create docker-compose.yml

Create this file in the AspNetCoreMicroservicesDemo root folder.

version: '3.9'

services:
  postgresdb:
    image: postgres:16
    container_name: postgresdb
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres123
      POSTGRES_DB: studentdb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  studentservice:
    build:
      context: .
      dockerfile: StudentService/Dockerfile
    container_name: studentservice
    environment:
      ASPNETCORE_ENVIRONMENT: Development
      ASPNETCORE_URLS: http://+:8080
    ports:
      - "5001:8080"
    depends_on:
      - postgresdb

  teacherservice:
    build:
      context: .
      dockerfile: TeacherService/Dockerfile
    container_name: teacherservice
    environment:
      ASPNETCORE_ENVIRONMENT: Development
      ASPNETCORE_URLS: http://+:8080
    ports:
      - "5002:8080"
    depends_on:
      - postgresdb

volumes:
  postgres_data:
Inside Docker Compose, service names such as postgresdb act like host names on the internal Docker network.

8. Run the lab

  1. Open a terminal in the solution root folder.
  2. Build and start the containers:
docker compose up --build
  1. Open the API endpoints in your browser or Postman.
http://localhost:5001/swagger
http://localhost:5002/swagger
  1. Stop the containers when finished:
docker compose down

9. How to verify everything works

  • Check whether the PostgreSQL container is running.
  • Open Swagger for each service.
  • Send a POST request to create a student.
  • Send a GET request to read the inserted data.
docker ps

docker compose logs studentservice

docker compose logs postgresdb
