Step 1: Prepare the Three VMs

Create three Kali Linux VMs: vpn-server - vpn-client and normal-client.

Adapter Setting Purpose
Adapter 1 NAT Internet access for installing packages.
Adapter 2 Host-only Network Communication between server and client VM.

Check network interfaces

Check this with all VMs 
ip -br a

Example output:

lo        UNKNOWN  127.0.0.1/8
eth0    UP       10.0.2.15/24
eth1    UP
Highlight note: In many VirtualBox setups, eth0 is NAT and eth1 is host-only/internal. Your interface name and ip addresses may be different.

Assign normal lab IPs

On the VPN Server VM:

sudo ip addr add 192.168.56.10/24 dev eth1
sudo ip link set eth1 up

On the VPN Client VM:

sudo ip addr add 192.168.56.20/24 dev eth1
sudo ip link set eth1 up

On the Normal Client VM:

sudo ip addr add 192.168.56.30/24 dev eth1
sudo ip link set eth1 up

From the client VMs, test normal connectivity:

ping -c 3 192.168.56.10
Important: Do not continue until the clients can ping 192.168.56.10. If this fails, the normal VM network is not working.

Step 2: Install WireGuard on Two VPN VMs (Server and Client)

Run this on both server and client:

sudo apt update
When it's done, run this. 
sudo apt install wireguard -y

Step 3: Generate WireGuard Keys

Each peer has a private key and public key. The private key stays on the machine. The public key is shared with the other peer.

Security reminder: Never share private keys publicly or commit them to GitHub.

On VPN Server VM

sudo -i
cd /etc/wireguard
umask 077
wg genkey | tee server_private.key | wg pubkey > server_public.key
cat server_public.key
exit

Display server private key when needed:

sudo cat /etc/wireguard/server_private.key

On VPN Client VM

sudo -i
cd /etc/wireguard
umask 077
wg genkey | tee client_private.key | wg pubkey > client_public.key
cat client_public.key
exit

Display client private key when needed:

sudo cat /etc/wireguard/client_private.key

Step 4: Configure the VPN Server

On the server VM:

sudo nano /etc/wireguard/wg0.conf

Add this:

[Interface]
Address = 10.10.0.1/24
ListenPort = 51820
PrivateKey = SERVER_PRIVATE_KEY_HERE

[Peer]
PublicKey = CLIENT_PUBLIC_KEY_HERE
AllowedIPs = 10.10.0.2/32
  • SERVER_PRIVATE_KEY_HERE = content of server private key.
  • CLIENT_PUBLIC_KEY_HERE = public key from client VM.
Explanation The server listens on UDP port 51820 and allows the client peer to use VPN IP 10.10.0.2.

Step 5: Configure the VPN Client

On the client VM:

sudo nano /etc/wireguard/wg0.conf

Add this:

[Interface]
Address = 10.10.0.2/24
PrivateKey = CLIENT_PRIVATE_KEY_HERE

[Peer]
PublicKey = SERVER_PUBLIC_KEY_HERE
Endpoint = 192.168.56.10:51820
AllowedIPs = 10.10.0.0/24
PersistentKeepalive = 25
  • CLIENT_PRIVATE_KEY_HERE = content of client private key.
  • SERVER_PUBLIC_KEY_HERE = public key from server VM.
Explanation: Endpoint tells the client where the server is on the normal network. AllowedIPs tells the client which destination traffic should go through the VPN tunnel.

Step 6: Start and Test the VPN

Start VPN on the server:

sudo wg-quick up wg0

Start VPN on the client:

sudo wg-quick up wg0

Check the VPN interface on both VMs:

ip addr show wg0

From the client VM:

ping -c 3 10.10.0.1

From the server VM:

ping -c 3 10.10.0.2

Check VPN status:

sudo wg

Look for latest handshake and transfer data.

Useful commands

Action Command
Stop VPN sudo wg-quick down wg0
Start VPN sudo wg-quick up wg0
Show VPN status sudo wg
Show routes ip route
Enable after reboot sudo systemctl enable wg-quick@wg0

Step 7: Create a Private Website on the VPN Server

The website will listen only on 10.10.0.1:8080, not on 192.168.56.10:8080.

Install Nginx

sudo apt update
sudo apt install nginx -y

Create website folder and homepage

sudo mkdir -p /var/www/private-vpn
sudo nano /var/www/private-vpn/index.html

Add this HTML:

<!DOCTYPE html>
<html>
<head>
    <title>Private VPN Website</title>
</head>
<body>
    <h1>Private VPN Website</h1>
    <p>You can see this page because you are connected through the VPN.</p>
</body>
</html>

Create Nginx site config

sudo nano /etc/nginx/sites-available/private-vpn

Add this:

server {
    listen 10.10.0.1:8080;

    server_name private-vpn.local;

    root /var/www/private-vpn;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
Important: listen 10.10.0.1:8080; means Nginx accepts website traffic only through the VPN IP address.

Enable and reload

sudo ln -s /etc/nginx/sites-available/private-vpn /etc/nginx/sites-enabled/private-vpn
sudo nginx -t
sudo systemctl reload nginx
If Nginx fails: start WireGuard first because 10.10.0.1 exists only after wg0 is active. Run sudo wg-quick up wg0, then test Nginx again.

Test from VPN Client

curl http://10.10.0.1:8080

Expected result includes:

<h1>Private VPN Website</h1>

Step 8: Test from the Normal Client VM

Try:

curl http://10.10.0.1:8080

Expected result:

No route to host
# or
Connection timed out

Also try:

curl http://192.168.56.10:8080

Expected result:

Connection refused
Explanation: The website is not listening on the server's normal IP. It is listening only on the VPN IP. Therefore, a machine must first join the VPN network before it can reach the website.
